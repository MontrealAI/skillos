# Autonomous RSI Cross-Domain Capability Transfer Atlas Proof

This proof tests the next critical SkillOS claim: whether verified capabilities transfer across domains.

A system that improves only inside one demo is not enough. SkillOS should turn traces into reusable skills, skills into releases, and releases into stronger performance on future work that was not used to tune the system.

## What it proves

Mechanism:

```text
demand → decomposition → capability atlas → specialist-agent market clearing → trace distillation → verifier courts → locked-domain transfer → release selection → routing policy → reinvestment → compounding generalization
```

## How to run

```text
GitHub → Actions → Autonomous RSI Cross-Domain Capability Transfer Atlas Proof → Run workflow
```

Recommended inputs:

```text
publish_to_repo: true
deploy_pages: false
```

Keeping `deploy_pages` false avoids GitHub Pages deployment collisions when the repository already has a Pages deployment workflow. The action still writes all generated files and refreshes the SkillOS command center.

## Boundary

This is a deterministic public benchmark and artifact generator. It does not claim achieved superintelligence, live revenue, investment advice, legal advice, policy advice, token recommendations, or Kardashev Type II civilization. It makes the capability-transfer mechanism measurable and repeatable.
