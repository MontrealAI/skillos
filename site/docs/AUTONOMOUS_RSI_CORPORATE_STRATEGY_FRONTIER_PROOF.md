# Autonomous RSI Corporate Strategy Frontier Proof

This package adds a fully autonomous GitHub Action proof for the corporate-strategy analogue of recursive self-improvement.

## Simple explanation

Recursive-style labs frame RSI as self-improving systems for knowledge discovery.

SkillOS tests the business version:

> Can a large specialist-agent organization recursively improve how it coordinates capital, compute, energy, data, trust, talent, product, distribution, validation, risk control, and reinvestment into compounding productive capability?

## What the Action does

The GitHub Action runs end-to-end without human review:

1. Generates a deterministic corporate-strategy benchmark.
2. Trains validation-gated RSI coordination releases.
3. Scores the final protocol on a locked holdout split.
4. Compares against a single corporate generalist, an uncoordinated multi-agent swarm, a static multi-agent committee, and a no-RSI large organization.
5. Runs negative controls: shuffled reward, random protocol, risk-blind coordination, and random strategy policy.
6. Computes bootstrap confidence intervals.
7. Writes a JSON proof receipt.
8. Writes a Markdown report.
9. Writes an executive proof webpage.
10. Updates the SkillOS homepage, registry, sitemap, and robots.txt.
11. Uploads artifacts.
12. Commits generated outputs back to the repository.
13. Deploys GitHub Pages.

## Public boundary

This is a deterministic benchmark proof using synthetic/redacted-style benchmark cases. It is not audited customer revenue, financial advice, investment advice, legal advice, achieved superintelligence, or achieved Kardashev Type II civilization.

It makes the corporate strategy mechanism publicly testable.

## Why it matters

The proof converts a grand thesis into a repeatable test:

- not just “agents are useful,” but whether a large coordinated agent organization beats weaker coordination structures;
- not just “RSI sounds powerful,” but whether validation-gated recursive releases improve a measurable protocol;
- not just “strategy should improve,” but whether improvements survive locked holdout evaluation, negative controls, and confidence intervals.
