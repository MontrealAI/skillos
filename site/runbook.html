What the automation guarantees

  • Scans GitHub workflow files and latest workflow runs.
  • Scans generated proof JSON, proof pages, proof docs, and badges.
  • Regenerates a fresh, public, user-friendly proof hub.
  • Commits generated site/status files back to the repository.
  • Deploys the static site to GitHub Pages.

Future-proofing rule

For any future proof workflow, include the word Proof or RSI in the workflow name, generate proof JSON into data/, and a visual proof page into site/. The command center will discover it automatically.

Safe claim boundary: The hub makes proof outputs visible and reproducible. It does not convert benchmark results into audited customer results or guarantees.
